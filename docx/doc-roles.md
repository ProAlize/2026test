# 文档职责与冲突仲裁（唯一入口）

## 目的

本文件用于解决 `plan-gpt.md`、`architecture-gpt.md`、`plan.md`、`exp.md`、`sum.md` 职责混淆问题。
后续协作请先看本文件，再决定修改目标文件。

## 文件职责矩阵

1. `plan.md`
   - 唯一职责：执行计划、里程碑、算力与交付物管理。
   - 负责：时间线、阶段退出条件、资源预算、任务优先级。
   - 不负责：具体模型结构参数、详细实验矩阵定义、文献结论裁定。

2. `exp.md`
   - 唯一职责：实验协议与评测规范（实验真值源）。
   - 负责：任务设定、训练/评测协议、主表和消融矩阵、指标与验收标准。
   - 不负责：项目排期、叙事风格、文献分级讨论。

3. `architecture-gpt.md`
   - 唯一职责：方法结构规范（架构真值源）。
   - 负责：模块设计、数据流、张量形状、训练/推理路径、MVP/完整版结构差异。
   - 不负责：实验资源排期、baseline 分层政策、投稿定位判断。

4. `sum.md`
   - 唯一职责：阶段性审计与文献快照（复盘文件）。
   - 负责：问题诊断、风险盘点、文献趋势总结、重构建议。
   - 不负责：作为当前实验配置或排期执行依据。

5. `plan-gpt.md`
   - 唯一职责：历史推导稿与备选方案池（草案文件）。
   - 负责：思路来源、候选设定、论证链备份。
   - 不负责：作为当前执行真值源。

## 冲突仲裁规则

1. 时间线/资源冲突：以 `plan.md` 为准。
2. 协议/指标/主表冲突：以 `exp.md` 为准。
3. 结构/实现细节冲突：以 `architecture-gpt.md` 为准。
4. `sum.md` 和 `plan-gpt.md` 与上述三者冲突时，视为历史建议，不覆盖执行规范。

## 更新流程

1. 先改真值源：`plan.md` 或 `exp.md` 或 `architecture-gpt.md`。
2. 再更新 `sum.md` 的审计结论（如需要）。
3. 最后在 `plan-gpt.md` 记录“保留为备选”或“已弃用”的历史信息。
